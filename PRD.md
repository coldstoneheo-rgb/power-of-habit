# [기획서] 앱 이름: 습관의 힘 (The Power of Habit)

## 1. 개요 및 디자인 콘셉트
* **개요:** 사용자가 일상적인 습관을 등록하고, 직관적인 체크와 다채로운 시각적 통계 피드백(점수, 히트맵, 스트릭)을 통해 지속적인 동기부여를 얻는 Android 앱입니다.
* **핵심 UI 정체성:** **"최소한의 요소로 최대한의 정보 시각화"**
  * 개발 시 UI 컴포넌트 간의 간격(Margin/Padding)을 넓게 잡고, 차트 컴포넌트 내부의 축 레이블이나 격자선을 최대한 숨김 처리(`visible: false`)하여 콘셉트를 완벽히 재현합니다.

### 1.1. 세부 디자인 가이드라인
1. **다크 모드 기반의 '포인트 컬러 시스템'**
   * **무채색과 유채색의 대비:** 배경은 완전한 블랙(#000000)에 가까운 깊은 다크 그레이 계열을 사용하여 시각적 피로도를 극대화로 낮춥니다. 텍스트와 핵심 아이콘에는 순백색(#FFFFFF)과 밝은 회색(#888888)을 매칭해 가독성을 높입니다.
   * **습관별 고유 아이덴티티:** '이불 개기(오렌지)', '수학 문제(스카이블루)' 등 습관마다 고유의 시그니처 컬러를 부여합니다. 이 컬러는 메인 화면의 점, 상세 화면의 그래프 선, 달력 하이라이트 등에 일관되게 적용되어 색상만으로 습관을 즉각 인지하게 합니다(Cognitive Parsing).
2. **경계선을 최소화한 '여백 중심(Whitespace-Driven) 레이아웃'**
   * **Box-less 디자인:** 항목을 구별할 때 굵은 테두리(Border) 대신, 미세한 명도 차이를 둔 레이어 분할이나 투명도(Opacity)가 적용된 얇은 구분선을 사용합니다.
   * **라운드 힌트(Rounded Corner):** 인풋 박스나 위젯 등은 모서리가 부드러운 스퀘어클(Squircle) 형태로 처리하여 트렌디하고 소프트한 감성을 더합니다.
3. **인포그래픽 스타일의 감성적 데이터 시각화**
   * **지수형 점수 그래프:** 꺾은선 차트에서 그리드 선을 과감히 제거하고 오직 트렌드 라인과 핵심 노드만 노출합니다.
   * **알약 형태(Pill-shaped)의 스트릭:** 연속 수행 기록 막대그래프는 끝이 둥근 캡슐 형태로 디자인하며, 기간에 따라 가로로 늘어납니다. 가장 긴 기록은 밝은 톤으로 하이라이트합니다.
   * **도트 매트릭스(Dot Matrix) 빈도 차트:** 요일과 월이 교차하는 지점에 성공 빈도가 높을수록 원의 크기(Radius)가 커지고 선명해지는 인터페이스를 제공합니다.
4. **모바일 홈 화면과의 유기적 연결 (위젯 디자인)**
   * **런처 내 디자인 일관성:** 1x1 컴팩트 위젯은 어두운 반투명 사각형 바탕 위에 단순한 마크만 띄워 홈 화면을 해치지 않게 합니다.
   * **글랜스 뷰(Glance View):** 캘린더 위젯은 텍스트를 빼고 '달력 격자'와 '테마 컬러 점'만 표시하여 즉각적인 피드백을 제공합니다.
5. **입력 스트레스를 낮추는 미니멀 폼(Form)**
   * **단순한 드롭다운과 텍스트 박스:** 화면 전체를 시원하게 쓰는 인풋 박스를 배치하고, 질문 양식을 직접 입력하게 유도합니다. 알림이나 빈도 설정은 터치 횟수를 최소화하는 드롭다운으로 제공합니다.

## 2. 데이터베이스(DB) 구조 설계 (SQLite / Room DB 스키마)

### 2.1. `Habits` (습관 마스터 테이블)
| 필드명 | 데이터 타입 | 설명 |
| :--- | :--- | :--- |
| `habit_id` | Integer (PK) | 습관 고유 ID (자동 증가) |
| `title` | String | 습관 이름 (예: 이불 개기) |
| `question` | String | 수행 여부를 묻는 질문 (예: 잠자리 정리 했나?) |
| `frequency_type` | String | 주기 종류 (매일 / 특정 요일 / 주 n회 / 월 n회) |
| `frequency_value`| String | 주기에 따른 세부 값 (예: 월, 수, 금) |
| `reminder_time` | Time | 알림 시간 (Null 허용, 끔 설정 가능) |
| `theme_color` | String | 위젯 및 텍스트에 적용할 헥사코드 (예: #FFCC00) |
| `habit_type` | String | 기록 형태 (단순 체크형 `CHECK` / 수치 입력형 `VALUE`) |
| `unit` | String | 수치 입력형일 때의 단위 (예: 쪽, kg, 회 - Null 허용) |

### 2.2. `HabitRecords` (습관 수행 이력 테이블)
| 필드명 | 데이터 타입 | 설명 |
| :--- | :--- | :--- |
| `record_id` | Integer (PK) | 기록 고유 ID |
| `habit_id` | Integer (FK) | `Habits` 테이블 참조 |
| `date` | Date | 수행 기준 날짜 (YYYY-MM-DD) |
| `status` | String | 수행 상태 (`COMPLETED`: 완료, `FAILED`: 미완료, `SKIPPED`: 건너뛰기) |
| `input_value` | Float | 수치 입력형일 때의 실제 기록 값 (Null 허용) |

## 3. 핵심 컴포넌트 기능 명세

### 3.1. 메인 대시보드 화면
* 오늘 기준으로 최근 4일간의 습관 달성 여부를 격자 형태로 보여주고 즉시 체크할 수 있는 화면입니다.
* 리스트의 날짜별 아이콘 단일 탭: '완료(O)', 다시 탭: '미완료(X)', 롱 탭: '건너뛰기(-)'.
* 수치형 습관은 탭 시 팝업을 통해 숫자를 입력받습니다.

### 3.2. 습관 상세 분석 화면
* 상단에 '한눈에 보기' 배치.
* 6가지 핵심 분석 위젯들이 스크롤 형태로 깔끔하게 배치.

### 3.3. 습관 등록/수정 화면
* 미니멀 폼 적용. 제목, 질문, 빈도 등 시원한 인풋 박스와 드롭다운 사용.
* 우측 상단 컬러 피커 칩을 통한 테마 컬러 지정.

## 4. 6대 핵심 위젯 컴포넌트 명세
(상세 분석 화면 및 홈 화면 위젯 모듈화용)

| 위젯 종류 | 시각적 형태 및 UI 구현 명세 | 데이터 연산 로직 |
| :--- | :--- | :--- |
| **1) 체크 위젯** | 홈 화면 1x1 칩 또는 리스트 아이템. 단일 탭 시 COMPLETED(체크 아이콘 활성화), 롱 탭 시 SKIPPED. 수치형은 팝업창. | 해당 날짜의 `HabitRecords` 상태 업데이트. |
| **2) 목표 위젯** | 주/월/분/연 단위 탭. 가로형 미니 프로그레스 바 UI. | 해당 기간 목표 대비 달성률(%) 시각화. |
| **3) 빈도 히트맵 위젯** | 세로(일~토), 가로(최근 6~12개월 월) 격자. 성공 빈도에 따라 원의 크기나 투명도가 진해지는 그리드 차트. | 특정 요일/월의 성공 빈도수 계산. |
| **4) 연속 수행 위젯** | '현재' 및 '최고' 연속 달성 일수를 가로형 막대그래프 칩 형태로 배치. 내부에 숫자와 날짜 표기. | COMPLETED 상태의 연속 일수 추적. |
| **5) 이력 위젯** | 컴팩트 월간 달력 그리드. 성공(테마색 원 채움), 실패(회색 작은점), 건너뛰기 등 상태 마킹. | 월 범위의 Date를 렌더링. |
| **6) 습관 점수 위젯** | 0~100% 스케일 꺾은선. 격자 숨김 처리. 지수 이동 평균(EMA) 로직 적용한 상승/하락 트렌드 라인. | 매일 달성 여부에 따른 EMA 알고리즘 계산. |
